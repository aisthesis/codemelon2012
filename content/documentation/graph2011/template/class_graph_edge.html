Class Overview

An edge is a simple immutable object consisting of a head and tail vertex. Edges do not directly support an associated edge weight. If a weight is needed, a WeightedEdge may be used instead. Edges may be inserted into a graph, and arrays of edges may be retrieved so that, for example, all edges in the graph can be examined or those edges can be identified whose head is some particular vertex.

The vertices of an edge may be represented as either integers or letters of the alphabet. Integers must be used for graphs having more than 26 vertices. If character representation is used, the edge should be constructed as in the example
Edge e = new Edge('r', 'u', true);
The toString() method of such an edge will then return the string "(r, u)".

Although edges are inherently directed, the same Edge objects are also used to describe edges of undirected graphs. An edge in an undirected graph is viewed as the equivalent of 2 directed edges between the corresponding vertices. More formally: An undirected graph contains the edge (from, to) iff it also contains the edge (to, from). In an undirected graph, however, these 2 directed edges will be counted as a single edge when counting the number of edges in the graph.

Public Constructors

public Edge (int from, int to)

Construct an edge from vertex from to vertex to using integer representation of vertices. The charRep() method of an edge constructed in this way will return false.

Throws
  • java.lang.IllegalArgumentException if the from or to argument is negative.

public Edge (int from, int to, boolean charRep)

Construct an edge from vertex from to vertex to using character representation of vertices if charRep is set to true or integer representation if charRep is false. The charRep() method of an edge constructed in this way will return the value of charRep.

If charRep is set to true, client programs will likely prefer to use characters in the specification of the head and tail vertices. For example: Edge e = new Edge('r', 'u', true);

Throws
  • java.lang.IllegalArgumentException if either of the following conditions holds:
    1. 1) the from or to argument is negative.
    2. 2) charRep is true and both from and to are neither in the range 'a' - 'z' nor 'A' - 'Z'. This includes the case where one vertex is in the range 'a' - 'z' and the other is in the range 'A' - 'Z'.

Public Methods

public boolean charRep ()

Determine whether or not the edge represents vertices as letters or as integers.

Returns
  • a boolean value showing whether or not the edge represents vertices as letters.

public boolean equals (Edge e)

Overrides the equals(Object o) method of java.lang.Object. 2 edges are considered to be equal iff 1) they have the same head and tail vertices and 2) they represent vertices in the same form (as letters or integers).

Parameters
e edge in reference to which the calling edge is to be tested for equality.
Returns
  • a boolean value showing whether or not both edges are to be considered equal.

public int from ()

Determine the edge's head vertex as an integer value. If the edge uses character representation, this value must be typecast to a char in order to immediately determine the character represented by the given integer.

Returns
  • an integer corresponding to the edge's head vertex.

public int to ()

Determine the edge's tail vertex as an integer value. If the edge uses character representation, this value must be typecast to a char in order to immediately determine the character represented by the given integer.

Returns
  • an integer corresponding to the edge's tail vertex.

public String toString ()

Overrides the toString() method of java.lang.Object. Returns a string of the form "(u, v)" representing the calling edge. If the edge represents vertices as characters (as determined by its charRep() method), u and v will be the corresponding letters. Otherwise, u and v will be the corresponding integers.

Returns
  • a string representation of the calling edge having the form "(u, v)".
